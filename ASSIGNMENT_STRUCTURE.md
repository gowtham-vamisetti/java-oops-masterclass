# Assignment Structure - Java OOP Masterclass\n\n## 📋 Assignment Format (UNIFORM & CONSISTENT)\n\nEvery assignment follows this exact structure:\n\n### Part 1: Problem Statement\n- Clear description of what to build\n- Requirements list\n- Example usage\n\n### Part 2: Base Classes/Interfaces (Provided)\n- Foundational code students should use\n- Clear documentation\n\n### Part 3: Student Task\n- Step-by-step requirements\n- Exact method signatures\n- Input/Output examples\n\n### Part 4: Solution Code (Reference)\n- Complete working solution\n- Inline comments explaining logic\n- Expected output\n\n### Part 5: Test Cases\n- Runnable test file\n- Multiple test scenarios\n- Validation examples\n\n---\n\n## File Naming Convention (CONSISTENT)\n\n```\nAssignment_[Level]_[Topic].java\nAssignment_[Level]_[Topic]_Solution.java\nAssignment_[Level]_[Topic]_Test.java\n```\n\n**Examples:**\n- `Assignment_1_MobilePhone.java` - Student task template\n- `Assignment_1_MobilePhone_Solution.java` - Complete solution\n- `Assignment_1_MobilePhone_Test.java` - Test cases\n\n---\n\n## Folder Structure (UNIFORM)\n\n```\n01-Classes-and-Objects/\n├── README.md\n├── CONCEPTS.md\n├── EXAMPLES.md\n├── assignments/\n│   ├── Assignment_1_MobilePhone.java\n│   ├── Assignment_1_MobilePhone_Solution.java\n│   └── Assignment_1_MobilePhone_Test.java\n└── run_tests.sh\n```\n\n---\n\n## All Assignments Will Follow This Pattern\n\n✅ Same structure for all 5 levels\n✅ Clear problem statement\n✅ Given code (base classes)\n✅ Clear requirements\n✅ Complete solution with comments\n✅ Test cases with expected output\n✅ Consistent file naming\n✅ Easy to understand progression\n"
